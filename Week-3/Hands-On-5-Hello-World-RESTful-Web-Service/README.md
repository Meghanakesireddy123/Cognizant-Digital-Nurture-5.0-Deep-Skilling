# Hands-On 5: Hello World RESTful Web Service

## Objective

The objective of this exercise is to develop a simple RESTful Web Service using Spring Boot that returns the text **"Hello World!!"**. The service is tested using both a web browser and Postman while verifying the application logs through SLF4J.

---

# Scenario

Develop a REST API that responds with a hardcoded message **"Hello World!!"** whenever a client sends an HTTP GET request to the `/hello` endpoint.

---

# REST API Details

| Property    | Value           |
| ----------- | --------------- |
| HTTP Method | GET             |
| URL         | /hello          |
| Controller  | HelloController |
| Method Name | sayHello()      |
| Return Type | String          |

---

# HelloController.java

```java
package com.cognizant.springlearn.controller;

import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {

    private static final Logger LOGGER =
            LoggerFactory.getLogger(HelloController.class);

    @GetMapping("/hello")
    public String sayHello() {

        LOGGER.info("START - sayHello()");

        String message = "Hello World!!";

        LOGGER.info("END - sayHello()");

        return message;
    }
}
```

---

# Explanation of the Code

### @RestController

Marks the Java class as a REST Controller. Every method inside this class can return data directly to the client.

---

### @GetMapping("/hello")

Maps the HTTP GET request to the `sayHello()` method.

When the user opens:

```
http://localhost:8083/hello
```

Spring Boot automatically invokes the `sayHello()` method.

---

### sayHello() Method

This method:

* Receives the client request.
* Logs the start of execution.
* Returns the hardcoded string **"Hello World!!"**.
* Logs the completion of execution.

---

### SLF4J Logging

```java
LOGGER.info("START - sayHello()");
LOGGER.info("END - sayHello()");
```

These log statements help verify that the controller method has been executed successfully.

# SME Questions and Answers

## 1. What happens when the URL `/hello` is accessed?

* The browser sends an HTTP GET request.
* Spring Boot receives the request.
* The request is mapped to the `sayHello()` method using `@GetMapping`.
* The method executes and returns **"Hello World!!"**.
* Spring Boot sends the response back to the client.

---

## 2. What are the HTTP Header details received?

Some common HTTP response headers are:

| Header         | Description                   |
| -------------- | ----------------------------- |
| Content-Type   | text/plain;charset=UTF-8      |
| Content-Length | Length of the response        |
| Date           | Date and time of the response |
| Connection     | keep-alive                    |

---

## 3. How to view HTTP Headers in Google Chrome?

1. Open Chrome.
2. Press **F12** to open Developer Tools.
3. Select the **Network** tab.
4. Refresh the page.
5. Click the **hello** request.
6. Open the **Headers** section to view Request Headers and Response Headers.

---

## 4. How to view HTTP Headers in Postman?

1. Open Postman.
2. Send a **GET** request to:

```
http://localhost:8083/hello
```

3. Click the **Headers** tab.
4. View Request Headers and Response Headers generated by the server.
